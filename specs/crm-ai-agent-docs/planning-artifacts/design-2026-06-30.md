## Adaptation rules (never copy CRM text)

Rewrite every doc for website's verified reality: pnpm@10.6.5 (not bun); Next.js 16 pages-router
(not RSBuild/React Router); React 19 / MUI 9 / TS 6; Apollo Client 4 + Apollo Server 5 +
react-hook-form + i18next (no Zustand/Redux/tsyringe/DI); bulletproof-react feature layout
`src/features/<feature>/...` with kebab-case + public-API barrels enforced by dependency-cruiser
(`src-feature-name-kebab-case`, `features-import-via-public-api`, `no-cross-feature-imports`, …);
tests under `src/test/{testing-library,unit,apollo-server,e2e,visual,load,memory-leak}` with
`TEST_ENV=client|server`, Playwright+Mockoon, faker, Stryker, K6, memlab, Lighthouse;
observability via @sentry/node + @sentry/react. Commands use real Makefile targets
(`make format`, `make lint` = lint-next+lint-tsc+lint-md+lint-deps, `make test-*`,
`make pr-comments`). The rust-code-analysis complexity gate (`make lint-metrics`,
`config/metrics-policy.json`) is delivered by sibling issue #224 and is referenced as the repo's
metrics policy for complexity-management / quality-standards parity.
